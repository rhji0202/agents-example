---
description: Generate Next.js page-level UI/UX spec from planning docs (PRD, ARCHITECTURE, PERMISSIONS, UI-GUIDE)
---

$ARGUMENTS를 페이지 이름으로 사용하여 ui-spec 스킬을 실행한다.

페이지 이름: $ARGUMENTS

ui-spec 스킬의 워크플로우를 따라 진행:
1. 기획 문서 (PRD, ARCHITECTURE, PERMISSIONS, UI-GUIDE) 읽기
2. 해당 페이지의 요구사항·권한·API 분석
3. docs/ui/{page}-ui.md 생성
4. 결과를 사용자에게 제시하고 피드백 요청
