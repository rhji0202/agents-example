# PackForward (배대지 플랫폼)

## Problem
해외 직구 소비자가 여러 쇼핑몰에서 상품을 구매할 때
각각 국제 배송하면 건당 $20~40의 배송비가 발생한다.
3건만 합치면 $30~60 절약 가능하지만,
개인이 현지 창고를 운영할 수 없다.

## Solution
미국 현지 창고 주소를 제공하고,
도착한 상품을 보관 → 합배송하여 한국으로 발송한다.
소비자는 웹에서 입고 확인, 합배송 신청, 결제, 추적을 한다.

## Users
| 유형 | 설명 | 핵심 니즈 |
|------|------|----------|
| 소비자 | 해외 직구 구매자 (월 1~5회) | 배송비 절감, 입고/배송 상태 확인 |
| 창고 관리자 | 미국 현지 창고 직원 | 빠른 입고 처리, 검수, 포장 |
| 운영 관리자 | 서비스 운영자 | 전체 현황 파악, 정산, 정책 관리 |

## Core Flow
1. 소비자가 가입하면 개인 미국 창고 주소를 받는다
2. 해외 쇼핑몰에서 구매 시 이 주소로 배송한다
3. 상품이 창고에 도착하면 입고 알림을 받는다
4. 원하는 상품을 묶어 합배송을 신청한다
5. 무게 기반 배송비를 결제한다
6. 한국행 발송 후 추적번호로 배송을 확인한다

## Goals
| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 입고→합배송 전환율 | 70%+ | stored 건 중 consolidated 비율 |
| 결제→발송 리드타임 | 24시간 이내 | paid_at → shipped_at 차이 |
| 월간 재이용률 | 40%+ | 월 2회 이상 결제 사용자 비율 |
| 가입→첫 결제 전환 | 50%+ | 가입 후 30일 내 첫 결제 |

## Scope
### v1 포함
- 미국 창고 1개소
- 합배송/재포장
- 무게/부피 기반 배송비 계산
- 카드 결제 (토스페이먼츠)
- 입고 사진 제공
- 배송 추적

### v1 제외
- 일본/독일 창고 — 운영 검증 후 확장
- 구매대행 — 배송대행만 (직접 구매 안 함)
- 관세 자동 납부 — 안내만 제공
- 모바일 앱 — 웹 반응형으로 대체
- 실시간 위치 추적 — 배송사 추적 페이지 링크로 대체

## Constraints
- 개인통관부호(P번호) 필수 — 한국 세관 통관 시 필요
- 150 USD 이하 면세 — 합배송 시 건당 신고가 기준
- 무료 보관 30일 — 이후 일별 보관료 부과
- Supabase 사용 — 별도 백엔드 인력 없음

## Risks
| 리스크 | 영향 | 대응 |
|--------|------|------|
| 배송사 API 장애 | 운송장 생성 불가, 발송 지연 | 수동 생성 fallback + 관리자 알림 |
| 환율 급변동 | 예상 배송비 ≠ 실제 결제 금액 | 결제 시점 환율 고정, 차액 흡수 |
| 통관 지연/보류 | 소비자 불만, CS 부하 | 보류 즉시 SMS+이메일 알림 + 서류 안내 |
| 창고 물량 폭주 | 입고 처리 지연 | 입고 예상 알림으로 사전 준비 유도 |
| 분실/파손 | 소비자 신뢰 하락 | 입고 검수 사진 필수, 클레임 프로세스 v2에서 |

## Open Questions

- 보관료 기산점은 inspected_at인가 stored_at인가?
- 금지품목 폐기 시 비용은 소비자 부담인가 서비스 부담인가?
- 합배송 묶음 내 상품별 신고가를 따로 받을 것인가, 총액만 받을 것인가?
- 창고 운영 시간 외 도착 택배의 처리 기준은?
