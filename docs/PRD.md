# PRD: Todo List App

## Executive Summary

개인용 할 일 관리 웹 애플리케이션. 할 일의 생성·완료·삭제를 빠르고 직관적으로 처리한다. Runway 파이프라인 검증을 겸한 MVP.

### Phase Guide

| Phase | 시점 | 기준 |
|-------|------|------|
| **Phase 0** | MVP | 핵심 CRUD + 필터링 |
| **Phase 1** | 런칭 후 30일 | 인증, 카테고리, 우선순위 |
| **Phase 2** | 스케일 단계 | 공유, 실시간 동기화 |

_Phase 표기가 없는 섹션은 Phase 0 (필수)_

## Problem

할 일을 빠르게 기록하고 관리할 수 있는 단순한 도구가 필요하다. 기존 Todo 앱들은 기능이 과도하거나, 회원가입이 필수이거나, 무거운 UI로 진입 장벽이 높다.

배경: 가볍고 빠른 할 일 관리 도구에 대한 수요. 회원가입 없이 즉시 사용 가능한 경험을 제공한다.

## Solution

브라우저에서 즉시 사용할 수 있는 할 일 관리 앱. 회원가입 없이 바로 시작하고, 할 일 추가·완료·삭제가 한 화면에서 가능하다.

## Stakeholders

| 역할 | 설명 | 핵심 니즈 | 접점 채널 |
|------|------|----------|----------|
| 사용자 | 할 일을 관리하는 개인 | 빠른 입력, 직관적 관리 | 웹 |

## User Journeys

### 사용자 Journey

1. 웹 앱에 접속한다
2. 입력 필드에 할 일을 입력하고 Enter 또는 추가 버튼을 누른다
3. 할 일 목록에 새 항목이 추가된다
4. 완료한 항목의 체크박스를 클릭하여 완료 처리한다
5. 필터(전체/진행중/완료)로 원하는 항목만 본다
6. 필요 없는 항목은 삭제 버튼으로 제거한다
7. 할 일 제목을 클릭하여 수정할 수 있다

### 예외 시나리오

| 시나리오 | 유형 | 영향 | 고객 경험 | 처리 방안 | Phase |
|----------|------|------|----------|----------|-------|
| 빈 텍스트 제출 | 입력 오류 | 무효 데이터 | 입력 필드 포커스 + 안내 | 프론트 검증 (빈 문자열 차단) | 0 |
| 네트워크 오류 | 시스템 | 데이터 미저장 | 에러 토스트 + 재시도 | 낙관적 업데이트 롤백 | 0 |
| DB 연결 실패 | 시스템 | 전체 서비스 | 에러 페이지 | 헬스체크 + 재시작 | 0 |

## Constraints & Risks

### Constraints

- 인증 없음 (Phase 0) — 모든 사용자가 같은 데이터를 공유
- 단일 서버 배포
- MariaDB Docker 로컬 실행

### Risks

| 리스크 | 확률 | 심각도 | 대응 |
|--------|------|--------|------|
| Next.js 16 breaking changes | 중 | 중 | node_modules/next/dist/docs/ 확인 |
| 인증 없이 데이터 공유됨 | 높 | 낮 | MVP 한정, Phase 1에서 인증 추가 |
