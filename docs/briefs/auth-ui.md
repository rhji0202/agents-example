# Auth UI — Admin

## 관리자 로그인

- Path: /admin/login
- Role: all staff

### Layout

```
┌────────────────────────────┐
│       HubNext 로고          │
│   ┌────────────────────┐   │
│   │ 아이디              │   │
│   └────────────────────┘   │
│   ┌────────────────────┐   │
│   │ 비밀번호            │   │
│   └────────────────────┘   │
│        [ 로그인 ]          │
└────────────────────────────┘
```

### Form Fields

| Field | Label | Type | Required | Validation | Placeholder |
|-------|-------|------|----------|------------|-------------|
| username | 아이디 | text | YES | 영문+숫자 4~20자 | "아이디 입력" |
| password | 비밀번호 | password | YES | 8자 이상 | "비밀번호 입력" |

### Actions

| Button | Condition | Action |
|--------|-----------|--------|
| 로그인 | 필드 모두 입력 | POST /auth/login → 대시보드 이동 |

### States

| State | 표시 |
|-------|------|
| 로딩 | 버튼 스피너 |
| 에러 | "아이디 또는 비밀번호가 올바르지 않습니다" 인라인 |

## 고객 목록

- Path: /admin/members
- Role: admin, customer_service

### Layout

```
[검색/필터 바]
  키워드 입력 + [유형 ▼] [상태 ▼] [등급 ▼] [검색]

[테이블]
  ┌────┬──────┬──────────┬────────┬──────┬──────┬──────┐
  │ □  │ 이름 │ 이메일   │ 전화   │ 유형 │ 등급 │ 상태 │
  ├────┼──────┼──────────┼────────┼──────┼──────┼──────┤
  │ □  │김민준│mj@..     │010-... │개인  │Silver│ 활성 │
  │ □  │(주)테크│info@.. │02-...  │사업자│Gold  │ 활성 │
  └────┴──────┴──────────┴────────┴──────┴──────┴──────┘

[하단] [ 내보내기 ] [ 일괄 알림 ]  ◀ 1 2 3 ▶
```

### Actions

| Button | Condition | Action |
|--------|-----------|--------|
| 검색 | - | 키워드+필터 조회 |
| 내보내기 | 선택 행 ≥1 | 엑셀 다운로드 |
| 일괄 알림 | 선택 행 ≥1 | 알림 발송 모달 |
| 행 클릭 | - | /admin/members/:id 이동 |

### States

| State | 표시 |
|-------|------|
| 로딩 | 테이블 스켈레톤 5행 |
| 빈 목록 | "조건에 맞는 고객이 없습니다" |
| 에러 | "목록을 불러올 수 없습니다" + 재시도 |

### Conditional

- 사업자 → 이름 컬럼에 상호명 표시

### Status Badge

| Status | Color | Label |
|--------|-------|-------|
| active | green | 활성 |
| pending | blue | 대기 |
| dormant | yellow | 휴면 |
| suspended | red | 정지 |
| deactivated | gray | 탈퇴 |
| rejected | orange | 반려 |

## 고객 상세

- Path: /admin/members/:id
- Role: admin, customer_service

### Layout

```
[헤더] 이름 + 유형 뱃지 + 상태 뱃지 + [상태 변경 ▼] [메시지 보내기]

[섹션: 기본 정보]
  이름 | 이메일 | 전화 | 가입일 | 최근 로그인 | 등급 [변경]

[섹션: 거래 요약]
  총 배송 건수 | 진행 중 | 총 결제 금액 | 미결제 | 환불 이력

[섹션: 포인트]  잔액: 12,500P  [수동 지급] [수동 차감]
  ┌────────┬──────┬───────┬───────┬──────────┐
  │ 날짜   │ 유형 │ 금액  │ 잔액  │ 사유     │
  └────────┴──────┴───────┴───────┴──────────┘

--- 사업자 전용 ---
[섹션: 사업자 정보]
  상호명 | 사업자번호 | 대표자 | 업태/업종
  담당자: 이름 | 연락처 | 이메일 | 부서

[섹션: 계약 조건]
  할인율 | 정산 주기 | 전용 담당자 | 무료 보관일 | 특별 서비스
  세금계산서: ON/OFF    [계약 수정]

[섹션: 서류] (pending 사업자만)
  서류 목록 + 미리보기  [승인] [반려]
```

### Actions

| Button | Condition | Action |
|--------|-----------|--------|
| 상태 변경 | admin | 드롭다운 선택 + 사유 입력 모달 |
| 등급 변경 | admin | 등급 선택 드롭다운 |
| 포인트 지급 | admin, cs | 금액+사유 입력 모달 |
| 포인트 차감 | admin, cs / 잔액>0 | 금액+사유 입력 모달 |
| 계약 수정 | admin / 사업자만 | 계약 편집 모달 |
| 서류 승인 | admin / pending만 | 확인 모달 → active 전환 |
| 서류 반려 | admin / pending만 | 반려 사유 입력 → rejected |

### States

| State | 표시 |
|-------|------|
| 로딩 | 섹션별 스켈레톤 |
| 에러 | "고객 정보를 불러올 수 없습니다" + 재시도 |

### Conditional

- individual → 사업자 정보·계약·서류 섹션 숨김
- business → 사업자 정보·계약 섹션 표시
- status=pending + business → 서류 섹션 표시
- point_balance=0 → 차감 버튼 비활성화

## 직원 상세

- Path: /admin/staff/:id
- Role: admin

### Layout

```
[헤더] 이름 + 사번 + 상태 뱃지 + [수정] [비밀번호 초기화]

[섹션: 기본 정보]  이름 | 이메일 | 사번 | 입사일
[섹션: 조직 정보]  부서 | 직급 | 담당 업무
[섹션: 계정 정보]  상태 | 최근 로그인 | 비밀번호 변경일

[섹션: 권한 설정]
  □ 관리자 (전체 권한)
  □ 물류관리 (보관/배송/랙관리)
  □ 고객관리 (고객/결제/문의)
  □ 운영관리 (게시판/통계/로그)
  □ 리포트 (조회/다운로드)
  [권한 저장]
```

### Actions

| Button | Condition | Action |
|--------|-----------|--------|
| 수정 | admin | 기본+조직 정보 편집 모드 |
| 비밀번호 초기화 | admin | 확인 모달 → 임시 비밀번호 발급 |
| 권한 저장 | admin | 체크박스 변경 저장 |
| 비활성화 | admin / active | 확인 모달 → inactive |
| 재활성화 | admin / inactive | active 전환 |

### States

| State | 표시 |
|-------|------|
| 로딩 | 섹션별 스켈레톤 |
| 에러 | "직원 정보를 불러올 수 없습니다" + 재시도 |

### Conditional

- status=inactive → "비활성 계정" 경고 배너 + 재활성화 버튼
- password_changed_at > 90일 → "비밀번호 변경 필요" 경고

## 직원 등록

- Path: /admin/staff/new
- Role: admin

### Layout

```
[섹션: 직원 정보]  이름* | 이메일* | 입사일
[섹션: 조직 정보]  부서* | 직급* | 담당 업무
[섹션: 초기 권한]  (부서 선택 시 기본 권한 자동 체크)
  □ 관리자 □ 물류관리 □ 고객관리 □ 운영관리 □ 리포트
[하단] [취소] [등록하기]
```

### Form Fields

| Field | Label | Type | Required | Validation | Placeholder |
|-------|-------|------|----------|------------|-------------|
| name | 이름 | text | YES | 2~50자 | "이름 입력" |
| email | 이메일 | email | YES | 이메일 형식, 중복 확인 | "email@hubnext.com" |
| hire_date | 입사일 | date | NO | 미래 불가 | "YYYY-MM-DD" |
| department | 부서 | select | YES | - | "부서 선택" |
| position | 직급 | select | YES | - | "직급 선택" |
| responsibilities | 담당 업무 | textarea | NO | 200자 | "담당 업무 입력" |

### Actions

| Button | Condition | Action |
|--------|-----------|--------|
| 등록하기 | 필수 입력 완료 | POST → 사번 자동 생성 → 상세 이동 |
| 취소 | - | 직원 목록으로 이동 |

### States

| State | 표시 |
|-------|------|
| 로딩 | 버튼 스피너 |
| 이메일 중복 | "이미 사용 중인 이메일입니다" 인라인 |
| 성공 | "직원 등록 완료. 사번: HN20260042" 토스트 |
| 에러 | "등록에 실패했습니다" + 재시도 |

### Conditional

- 부서 선택 시 기본 권한 자동 체크:
  - 물류팀 → logistics
  - 고객지원팀 → customer_service
  - 운영팀 → operations
