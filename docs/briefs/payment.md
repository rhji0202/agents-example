# Payment Brief

## Summary

측정 데이터 기반으로 배송비를 계산하고, 고객에게 결제를 요청·확인한다. 차등 요금제와 예치금을 지원한다.

## Flow

1. 모든 상품 측정 완료 → 시스템이 배송비 자동 계산
2. 비용 구성: 기본 배송비 + 부가서비스(재포장/보험/사진) + 세금
3. 개인/셀러 차등 요금 적용
4. 고객에게 결제 요청 알림 (72시간 유효)
5. 고객이 결제 수단 선택 (카드/계좌이체/예치금)
6. 결제 성공 → 출고 대기
7. 결제 실패 → 24시간 내 3회 자동 재시도
8. VIP 고객 결제 유예 24시간 가능
9. 72시간 내 미완료 시 주문 자동 취소

## Business Rules

- 배송비 = max(실제 무게, 부피 무게) 기준 계산
- 개인/셀러 요금표 별도 관리
- 결제 수단: 카드(PG사), 계좌이체, 예치금
- 예치금 잔액 부족 시 다른 수단으로 전환
- 결제 유효기간 72시간, 초과 시 자동 취소
- 결제 실패 시 24시간 내 3회 자동 재시도 (PG사 정책 준수)
- VIP 고객 결제 유예 24시간 (재무팀 승인)
- 결제 완료 시 세금계산서/현금영수증 발행 준비

## Status

### 결제 요청

```
pending → paid
       ↘ failed → retrying(3회) → paid
                                 ↘ failed → cancelled(72h 초과)
```

### 예치금

```
charged(충전) → used(사용) → refunded(환불)
```

## Screens

| Screen | Path | Key Features |
|--------|------|-------------|
| 결제 (고객) | /mypage/orders/:id/pay | 비용 명세, 결제 수단 선택, 예치금 잔액 |
| 예치금 관리 (고객) | /mypage/deposit | 잔액, 충전, 사용 내역 |
| 결제 관리 | /admin/payments | 결제 대기/완료/실패 필터, 금액 정렬 |
| 결제 상세 | /admin/payments/:id | 비용 명세, 결제 이력, 환불 처리, VIP 유예 승인 |
| 요금 설정 | /admin/settings/pricing | 개인/셀러 요금표, 부가서비스 요금 |
